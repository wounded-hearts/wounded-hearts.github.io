ACL-访问控制列表

一、概述

  1. 全称：Access Control List（访问控制列表 ）
  2. 作用：ACL是一种包过滤技术
  3. ACL基于IP包头的IP地址、四层TCP/UDP头部的端口号、[五层数据]（ 基于三层和四层过滤）
  4. 位置：ACL在路由器上配置，也可以在防火墙上配置（一般称为策略）
  5. ACL主要分为2大类：
    • 标准ACL：standard
    • 扩展ACL：extended
  6. ACL表格式：
    • access-list 表号 动作 条件 （表号：决定了ACL的类型；动作：允许/拒绝）
    • ip access-list standard/extended 自定义表名 + 动作 条件

二、ACL原理

  1. ACL表必须应用到路由器接口的进或出方向才能生效
  2. 一个接口的一个方向只能应用一张表
  3. 进还是出方向应用？取决于流量控制的总方向
  4. ACL表是严格自上而下检查每一条，所以要注意书写顺序
  5. 每一条策略是由条件和动作组成，当某流量满足上一条的策略时，会忽略下面的策略；当某流量不满足上一条的策略时，才会继续检查下一条策略
  6. 标准ACL尽量写在靠近目标的地方；扩展ACL尽量写在靠近源端的地方
  7. 小技巧：
    • 做流量控制，首先要判断ACL写的位置（哪个路由器？哪个接口的哪个方向？）
    • 再怎么考虑怎么写ACL
    • 如何写？
      • 首先要判断最终要允许所有还是拒绝所有
      • 然后写的时候要注意：将严格的控制写在前面
  8. 注意事项：
    • 一般情况下，标准ACL或扩展ACL一旦编写好，就无法删除ACL策略、修改ACL策略的顺序，只能一直在最后添加新的策略
    • 如想修改或插入或删除标准ACL和扩展ACL中的策略，只能删除整张表，重新写
    • 想要任意修改ACL表中的策略，必须使用命名ACL方式

三、标准ACL

  1. 表号：1 ~ 99

  2. 特点：只能基于源IP对包进行过滤

  3. 命令：config t

    • 添加ACL表：access-list 表号 permit/deny 源IP/源网段 反子网掩码 （permit允许；deny拒绝）
    • 删除ACL表：no access-list 表号
    • 查看ACL表：show ip access-list [表id] （不写表id，默认显示所有的表）

  4. 将ACL表应用到接口：

  5. 反子网掩码：将正子网掩码0和1倒置示例如下：

    • 255.0.0.0 – 0.255.255.255
    • 255.255.0.0 – 0.0.255.255
    • 255.255.255 – 0.0.0.255

  6. 反子网掩码

    • 作用：用来匹配条件，与0对应的需要严格匹配，与1对应的忽略
    • 示例1：access-list 1 deny 10.0.0.1 0.255.255.255 （解释说明：该条目用来拒绝所有源IP为10开头的主机）
    • 示例2：access-list 1 deny 10.0.0.5 0.0.0.0（解释说明：该条目用来拒绝所有源IP为10.0.0.5的主机）
    • 简写示例2：access-list 1 deny host 10.0.0.5
    • 示例3：access-list 1 deny 10.0.0.5 255.255.255.255（解释说明：该条目用来拒绝所有的主机）
    • 简写示例3：access-list 1 deny any

五、扩展ACL

  1. 表号：100 ~ 199
  2. 特点：可以基于源IP、目标IP、端口号、协议等对包进行过滤（过滤机制更加强大）
  3. 命令：
    • 格式：acc 100 permit/deny 协议 源IP/源网段 反子网掩码 目标IP/源网段 反子网掩码 [eq 端口号]（端口号为可选项）
  4. 协议：tcp/udp/icmp/ip
  5. 案例：
    • access-list 100 permit tcp host 10.0.0.1 host 20.0.0.3 eq 80（说明：只允许主机10.0.0.1访问20.0.0.3的80端口）
    • acc 100 deny ip host 10.0.0.1 20.0.0.0 0.0.0.255 （说明：拒绝主机10.0.0.1访问20网段）
    • acc 100 permit ip any any （说明：允许任意IP访问任意网段）

六、命名ACL

  1. 说明：命名ACL并不属于ACL类型，只是方便管理ACL

  2. 作用：可以对标准ACL或扩展ACL进行自定义命名

  3. 优点：

    • 自定义命名更容易辨认，也便于记忆
    • 可以任意修改ACL表，包括删除某一条策略、插入某一条策略

  4. 命令：config t

    • 创建命名ACL：ip access-list standard/extended 自定义表名（此时已进入命名ACL编辑模式）
    • 设置命名ACL表：动作 + 条件
    • 退出命名ACL编辑模式：exit
