数据链路层与交换机的工作原理

一、数据链路层

  1. 位置：位于OSI模型的第二层

  2. 传输单元：帧

  3. 帧结构，如下图所示

  4. 一个帧的最大容载量：6 + 6 + 2 + 1500 + 4 = 1518字节（仅限国内）

  5. 帧格式：

    • 有线传输：802.3（802代表国际有名的802个科学家）
    • 无线传输：802.11

  6. 帧头内容：目标MAC地址 + 源MAC地址（本机）

  7. 帧头中的类型的作用：识别上层协议，为上层提供服务

  8. MAC地址：

    • MAC地址 = 厂家标识 + 该厂家网卡的唯一标识（全球唯一）
    • MAC地址也叫做物理地址
    • 长度48位，6字节（8bit=1byte）

二、交换机的工作原理

  1. 工作在数据链路层的设备：交换机 / 网卡

  2. 交换机不能隔离网段，必须连接同一网段；路由器必须要隔离网段，即每个接口的网段必须不一致

  3. 交换机工作原理简图1：

  4. 交换机工作原理简图2：

  5. 交换机工作原理说明：

    • 交换机收到一个数据帧后
    • 首先学习帧中的源MAC地址来形成MAC地址表
    • 然后检查帧中的目标MAC地址，并匹配MAC地址表，如表中有匹配项，则单播转发；如表中无匹配项，则除接收端口外广播转发
    • MAC地址表的老化时间（超时）默认是300秒（可修改）

  6. 交换机的端口：

    • E：10Mb
    • F：100Mb
    • G：1000Mb
    • Te：10000Mb
    • 例如：F0/2：（F百兆接口；0模块号；2该模块中的接口号）

  7. 交换机接口速率自适应：1000/100/10M自适应；速率工作模式可以为10，100，1000任何一种状态

  8. 交换机端口状态：up / down（开启/关闭）

  9. 交换机端口处于关闭状态的3中可能：

    • 人工关闭端口
    • 端口速率不匹配
    • 双工模式不匹配 （老式交换机设备中的模式，比较落后，但是新版的也有，为了兼容）

  10. 交换机的双工模式说明：

    • 单工模式：只能打或只能听
    • 半双工模式：一个打一个听，需要等待对方说完才能说
    • 全双工模式：打字对骂，无需等待对方说完
    • 新版的交换机中，有半双工和全双工的自适应设置，若自适应没打开，接口就会不通

三、交换机基本工作模式

用户模式

  • 说明：进入设备的命令窗口，就进入了用户模式

  • 作用：可以查看交换机的基本简单配置，且不能做任何修改配置！

特权模式

  • 说明：在用户模式下，输入enable命令，可进入特权模式

  • 作用：可以查看所有的配置，但不能修改配置，可以做测试、保存、初始化操作！

全局配置模式

  • 说明：在特权模式下，输入configure terminal命令，可进入全局配置模式

  • 作用：默认不能查看配置，可以修改配置，且全局生效！

接口配置模式

  • 说明：在全局配置模式或者console口模式下，输入interface + 接口号，可进入指定的接口配置模式

  • 作用：默认不能查看配置，可以修改配置，且仅对该接口生效！

console口 / 线 / 控制台模式

  • 说明：在全局配置模式或接口配置模式下下，输入line console 0，可进入console口模式

  • 作用：默认不能查看配置，可以修改配置，且仅对console口生效！

四、交换机基本命令

  • 重要说明：

    1. 交换机中的命令，不需要完整输入，只要输入的命令可以唯一匹配即可；例如：exit可以输入ex即可

    2. exit和end命令，只输入e是不行的，因为不能唯一匹配，但是可以输入e?查看相关命令

其他命令

  1. 模式退出

    • exit：任意模式下，模式退出一级
    • end：直接退回到特权模式

  2. 重新加载设备配置：reload （特权模式下输入，若没有在磁盘的配置文件中，保存密码，重新加载后，恢复初始化状态，密码就不存在了，可以无密码进入用户模式）

  3. 取消命令解析：Switch(config)#no ip domain-lookup （进入全局配置模式后输入，不管在哪种模式下，输入错误的命令，会快速报错，否则会超时等待很长很长的时间，不利于实验）

增加命令

  1. 设置用户密码（只有明文方式）：

    • 作用：进入用户模式时，必须输入正确的用户密码才能进入，提高设备的安全性

    • password + 任意密码 （必须先进入console口模式）

  2. 设置特权密码（明文）：

    • 作用：进入特权模式时，需要输入正确的特权密码才能进入，提高设备的安全性

    • enable password + 密码 （必须先进入全局配置模式）

  3. 设置特权密码（密文）：

    • 作用：进入特权模式后，使用show + 配置文件名，只能看到密码的hash值，提高设备的安全性

    • 命令：enable secret + 密码（必须进入全局配置模式下）

    • 密码选择：当明文与密文同时存在时，优先选择密文密码，明文密码会失效，但是在配置文件中依然存在（5代表MD5）

  4. 进入模式时开启密码验证（先设置密码）：

    • login （在设置完密码后，输入login即可启动）
    • 开启验证后，再次进入用户模式时，必须输入正确的密码

删除命令

  1. 清除 / 初始化配置文件：erase（必须在特权模式下输入）

  2. 删除指定的配置：no + 命令

    • 在哪个模式下配置的，就在哪个模式下删除

    • 命令前面 + no即可完成删除/撤销

    • 原命令中有参数，并且参数具有唯一性，则删除时不需要加参数

修改命令

  1. 设备重命名：

    • hostname + 任意名称 （必须先进入全局配置模式，命令完成后立即生效）

  2. 保存设备上的配置：

    • 高级交换机中，存在运行内存和磁盘
    • 设备第一次开机后会在运行内存中生成running-config文件，关闭设备后此文件会一并被清除；磁盘中默认没有文件
    • 设备再次开机后，首先会在磁盘中查找startup-config文件，若磁盘中存在此文件，则复制此文件，并重命名为running-config，并放置在运行内存中进行读取；若磁盘中不存在startup-config文件，则设备会自己生成初始化文件running-config
    • copy running-config startup-config （在特权模式下，输入此命令，将设备上的所有配置保存在startup-config磁盘上，下次开机自动加载配置）
    • 命令简写：write或者wr （此命令时copy running-config startup-config的简写）

  3. 手动关闭指定接口：

    • interface f0/20：进入指定接口（在全局配置模式下输入）
    • shutdown：关闭指定接口

  4. 手动开启指定接口：

    • interface f0/20：进入指定接口（在全局配置模式下输入）
    • no shutdown：开启指定接口

  5. 强制执行特权命令：

    • do + 任意特权命令 （除特权模式外，其他模式也可以用特权命令，用户模式除外）
    • 全局配置模式：Switch(config)# do show ip interface brief （查看接口状态列表）
    • 接口配置模式：Switch(config-if)# do show ip interface brief或者do wr或者do show startup-config
    • console口模式：Switch(config-line)# do show ip interface brief

查询命令

  1. 说明：以下命令必须进入特权模式才能输入

  2. 查看设备的配置文件：show startup-config（查看磁盘上的配置文件，强烈建议，先将running-config保存为startup-config，进入配置文件按空格翻页）

  3. 查看交换机的MAC地址表：show mac-address-table

  4. 查看交换机的接口状态列表：show ip interface brief

五、路由器的使用

路由器概述

  1. 说明：路由器命令与交换机命令几乎一致

  2. 路由器在出厂时，默认关闭了接口，当线缆连接路由器后，不能直接连通，需要用命令打开

  3. 设备连接如下图所示：

连通路由器

  1. 打开路由器的管理终端，选择yes执行自动配置（傻瓜式配置），选择no自定义配置

  2. 进入指定接口f0/0，查看接口状态

  3. 使用命令no shutdown打开指定接口，注意保存配置write（特权模式）或者do write（其他模式，除用户模式外）

  4. 给路由器接口配置网关地址ip addr + IP + 子网掩码（注意在接口配置模式下输入）

  5. 给PC配置同一网段的IP，并测试连接

远程控制路由器

telnet连接

  1. 进入路由器的管理终端，切换到全局配置模式，输入命令：line vty 0 4 （0代表接口号，4代表同时远程控制路由器的设备数量）

  2. 设置成功后，用PC远程登陆telnet 10.0.0.154 ，在没有设置特权密码时，只能登陆用户模式，特权模式及其他模式无法进入

  3. 测试远程命令，修改路由器名称hostname sw0-888

ssh连接

  1. 打开路由器的管理终端，进入全局配置模式，开启虚拟终端line vty 0 4（路由器默认开启telnet服务），所以需要自己修改服务类型：transport input ssh

  2. 设置远程登陆密码（telnet连接时已设置过远程登陆密码和特权密码了，并保存了配置，所以此时可省略；若没设置过密码，必须重新设置）

  3. 在全局配置模式下，给路由器配置任意域名（重要操作）ip domain-name + 任意域名

  4. 生成ssh密钥crypto key generate rsa ，并远程登陆（登陆命令仅限于此模拟软件）

  5. 真实路由器ssh连接命令：ssh 用户名@ + IP

telnet与ssh同时连接

  1. 只需在路由器的管理终端中，进入虚拟终端，设置服务类型为all即可

  2. 在PC中，用telnet或者ssh都可以连接了

添加身份验证

  1. 作用：每次远程登陆，都需要输入账户和密码才能进入，进入后的各种操作，都会记录到日志中，方便追究问题

  2. 对上面的连接进行补充

  3. 进入路由器的管理终端，在全局配置模式下，添加账户与密码列表username + 账户 password + 密码，然后进入虚拟终端配置模式开启本地身份验证

六、交换机与路由器组合使用

  • 最终效果图

连通整个网络

  1. 进入路由器的管理终端，开启两个网络接口fa0/0和fa0/1，并添加两个网关10.0.0.254和20.0.0.254

  2. 打开所有的PC，配置对应的IP、子网掩码、网关（配置网关可跨网段访问设备）

  3. 检查网络环境，确认两个网段之间可互通

远程控制路由器

  • 进入路由器的管理终端，添加路由器的本地账户，开启虚拟终端，开启telnet服务，开启身份验证，添加特权密码

远程控制交换机一

  1. 进入交换机一的虚拟主机的管理终端，配置IP和子网掩码

  2. 进入交换机的管理终端，添加本地账户和密码，开启telnet服务，开启身份验证，设置特权密码，验证远程连接

远程控制交换机二

  1. 进入交换机二的管理终端，设置特权密码，添加本地账户密码；进入远程连接虚拟终端，开启远程服务，开启本地身份验证；重命名交换机

  2. 进入交换机二的虚拟主机的管理终端，配置IP和子网掩码，并开启此接口，进入全局配置模式，添加网关

  3. 使用PC机10.0.0.1，远程登陆交换机二
