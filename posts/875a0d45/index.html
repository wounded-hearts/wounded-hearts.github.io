一、用户管理

用户概述

  1. 说明：

    • 每一个用户登录系统后，拥有不同的操作权限
    • 每个账户都有自己唯一的SID（安全标识符）
    • SID全称：S-1-5-21-426206823-2579496042-14852678-500
    • 系统SID：1-5-21-426206823-2579496042-14852678
    • 用户UID：500
    • Windows系统管理员administrator的UID是500
    • 普通用户的UID从1000开始
    • 不同的账户拥有不同的权限，为不同的账户赋权限，也就是为不同账户的SID赋权限

  2. 查看SID命令：whoami /user

  3. Windows账户密码存储位置：C:\windows\system32\config\SAM

  4. Windows系统上，默认密码最长有效期42天（可自定义有效期）

内置账户

  1. 给人使用的账户：

    • 管理员账户：administrator
    • 来宾账户：guest
    • 普通用户：user

  2. 给计算机使用的账户：

    • 系统账户 == 权限至高无上：system
    • 本地服务账户 == 权限等于普通用户：local services
    • 网络服务账户 == 权限等于普通用户：network services

  • 每个用户都有自己的配置文件（家目录）在用户第一次登录时自动产生
  • 路径：C:\用户\（win7 / win8 / win10）；C:\Documents and Settings\（xp / win2003）

创建隐藏账户

  1. 第一步：在DOS窗口创建新用户new，并将其加入超级管理员组

    • net user news$ 123 /add （$表示在DOS窗口隐藏，此时创建的账户可在计算机管理中看到）

    • net localgroup administrators news$ /add

  2. 第二步：展开注册表HKEY_LOCAL_MACHINE \ SAM \ SAM（默认情况下这个项里没有任何内容，这是因为用户对它没有权限，右键SAM目录为administrator赋予完全控制权限）

  3. 第三步：先关闭注册表（让第二步功能生效），再打开注册表HKEY_LOCAL_MACHINE \ SAM \ SAM \ SAMDomains \ Account \ Users \ Names里面显示了系统当前存在的所有账户，选中news$, 在其右侧有一个名为“默认”，类型为”0x3ee”的键值。其中的“3ee”就是news$用户SID的结尾，即RID (这里使用十六进制表示）；在Users目录下有一个以“3EE”结尾的子项，这两个项里都是存放了用户news$的信息；在这两个项上单击右键，执行“导出”命令，将这两个项的值分别导出成扩展名为xxx.reg的注册表文件

  4. 第四步：打开DOS窗口，删除news$账户：net user news$ /del，重新打开注册表、计算机管理，确认已删除此账户

  5. 第五步：双击运行刚导出的news.reg和news_name.reg文件，再次打开注册表、计算机管理，确认在注册表中显示此账户、计算机管理中不显示；使用这个隐藏账户news$（注意添加$）可以登录系统，但缺点是仍然会产生用户配置文件，下面再对这个账户进行处理，进一步深度隐藏

  6. 第六步：打开注册表，找到administrator用户 的RID值“1f4”，展开对应的“000001F4“项

    点击000001F4其右侧有一个名为F的键值，这个键值中就存放了用户的SID；下面将这个键值的数据全部复制，并粘贴到000003EE项的F键值中，也就是将administrator用户的SID复制给了news$，这样在操作系统内部，实际上就把news$当做是administrator，此时的news$成了administrator的影子账户，与其使用同一个用户配置文件，news$账户也就被深度隐藏了

  7. 第七步：检查DOS窗口、计算机管理，确认不显示此账户news$，最后注销当前系统，登陆news$账户，密码123

  8. 注意事项：

    • 成功创建影子账户后，不管是在哪个账户操作，其操作痕迹都会在另一个账户中出现（因为使用同一个配置文件），例如：创建的文件及文件夹（注意隐藏）
    • 影子账户在regedit（注册表）中显示，不能删除，所以，可反向检查系统安全
