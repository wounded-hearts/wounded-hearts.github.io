有趣的Cisco Packet Tracer
发表于 2020-10-29| 更新于 2020-10-29|网络安全
字数总计:818|阅读时长: 2 分钟
|阅读量:

一、概述

  • Cisco Packet Tracer 是由Cisco公司发布的一个辅助学习工具，为学习思科网络课程的初学者去设计、配置、排除网络故障提供了网络模拟环境。用户可以在软件的图形用户界面上直接使用拖曳方法建立网络拓扑，并可提供数据包在网络中行进的详细处理过程，观察网络实时运行情况。可以学习IOS的配置、锻炼故障排查能力。

二、功能简介

添加与删除设备

  1. 打开软件，选择设备栏中的指定设备类型，然后选择某种设备，最后直接拖拽到空白区域即可

  2. 在右侧工具栏，点击选择工具，选中设备，可自定义设备位置

连接设备

  1. 多个设备之间，可用不同的线缆连接，实现互相通信

  2. 选择指定线缆 >> 点击设备 >> 选择插口，再次点击2处的线缆，可取消选择的线缆

  3. 一端的插口接好后，用鼠标将线缆移动到另外一台设备上，选择插口；至此，两台设备已相互连通

  4. 重复以上操作，连接多台设备，组成一个简单的内网环境，插口处的橙色代表未连通，等待一会变为绿色（模拟真实环境的延迟状态），代表相互连通 ，连接后的设备也可自定义位置

显示设备状态

  • 自定义显示的内容

给设备添加模块

  1. 当路由器插口不够用时，可以额外再添加插口，作为另外一个模块

  2. 点击路由器，选择要添加的模块（此时需要百兆插口），拖拽到设备的额外添加区域即可

  3. 选择线缆，点击路由器设备，此时的插口处有4个百兆接口，完全够用了

  4. 成功完成连接

三、模拟网络服务

模拟配置静态IP

  1. 点击服务器设备，选择Desktop，再选择IP图标，给服务器配置静态IP：10.0.0.1，完成后关闭窗口即可

  2. 点击PC设备，同上一步操作，配置静态IP：10.0.0.2

  3. 点击PC设备，在Desktop窗口中，选择命令窗口

  1. 执行终端命令，模拟网络状态

模拟web服务

  1. 点击服务器设备，选择services选项，选择HTTP服务，

  2. 点击edit按钮，编辑首页内容

  3. 打开PC设备，选择浏览器

  4. 在浏览器窗口输入网站地址：10.0.0.1，可访问模拟网站，https请求也可以

模拟DNS服务

  1. 点击服务器设备，选择services选项，选择DNS服务，添加完整域名

  2. 点击PC设备，在Desktop选项中，选择IP窗口，配置DNS

  3. 点击PC设备，在Desktop选项中，打开web浏览器，输入网站的完整域名，实现域名解析功能

模拟DHCP服务

  1. 点击服务器设备，选择services选项，再选择DHCP服务，配置作用域，注意选择on开启此服务

  2. 点击PC设备，在Desktop选项中打开IP窗口

  3. 选择DHCP选项，等待一小会儿（模拟真实环境的延迟状态）
